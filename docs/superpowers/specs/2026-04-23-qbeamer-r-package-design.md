# qbeamer R Package Design Spec

## Overview

Transform the qbeamer Quarto Beamer template into a full R package that provides:
1. A Quarto extension format (`qbeamer-beamer`)
2. RStudio project and document templates
3. Helper functions for rendering and previewing

## Quarto Extension

`_extension.yml` registers the format:

```yaml
title: qbeamer
author: Gabriel Cabrera Guzmán
version: 0.1.0
contributes:
  formats:
    beamer:
      include-in-header:
        - preamble.tex
      template-partials:
        - title.tex
      pdf-engine: xelatex
      navigation: horizontal
      theme: default
      colortheme: default
      aspectratio: 43
      keep-tex: true
```

The existing `before-body.tex` (renamed to `preamble.tex` — the content is LaTeX preamble, not body content) and `title.tex` move into `inst/extdata/_extensions/qbeamer/` as the single source of truth. Users reference the format as `format: qbeamer-beamer` in their QMD YAML.

**Note:** Remove `\usepackage{lipsum}` from `preamble.tex` — it is a demo-only package for filler text and should not ship in the extension.

## R Functions

### `install_extension(path = ".", overwrite = FALSE)`

Copies `_extensions/qbeamer/` from the installed R package into the target project directory. Creates the `_extensions/` folder if needed. Warns if already installed unless `overwrite = TRUE`.

### `create_project(path, ...)`

RStudio project template binding function. Called by RStudio when the user creates a new project from the template. It:
- Creates the project directory at `path`
- Copies `skeleton.qmd` into it
- Calls `install_extension(path)` to deploy the extension files
- Writes the `.Rproj` file

### `qbeamer_render(input, ...)`

Thin wrapper around `quarto::quarto_render()`. Checks that the extension is installed in the project before rendering. Passes `...` through.

### `qbeamer_preview(input, ...)`

Thin wrapper around `quarto::quarto_preview()`. Same pre-check for extension installation.

### `.onAttach()` (in `zzz.R`)

Displays a startup message with the package version when loaded.

**Exported functions:** `install_extension`, `create_project`, `qbeamer_render`, `qbeamer_preview`. NAMESPACE is auto-generated by roxygen2.

## RStudio Integration

### Project Template

Location: `inst/rstudio/templates/project/`

Appears in **File > New Project > New Directory > Quarto Beamer Presentation (qbeamer)**.

`qbeamer.dcf` contents:

```dcf
Binding: create_project
Title: Quarto Beamer Presentation (qbeamer)
OpenFiles: skeleton.qmd
Parameter: title
Widget: TextInput
Label: Presentation Title
Default: Untitled Presentation
```

### Document Template

Location: `inst/rmarkdown/templates/qbeamer-presentation/`

Appears in **File > New File > R Markdown > From Template > Quarto Beamer (qbeamer)**. Creates a single `skeleton.qmd` with minimal YAML front matter. User needs to run `qbeamer::install_extension()` to get the extension files.

`template.yaml` contents:

```yaml
name: Quarto Beamer (qbeamer)
description: A Quarto Beamer presentation using the qbeamer template.
create_dir: false
```

### Skeleton QMD

```yaml
---
title: "Untitled Presentation"
author:
  - name: "Author Name"
format:
  qbeamer-beamer: default
---

## First Slide

Content here.
```

## Example

The current `example.qmd` (E-M algorithm demo) moves to `inst/examples/example.qmd` as a non-vignette example, avoiding the need for a Quarto vignette engine at `R CMD check` time. Users can access it via `system.file("examples", "example.qmd", package = "qbeamer")`.

## Decisions

- **Package name:** qbeamer
- **Format reference:** `qbeamer-beamer` (Quarto extension style)
- **License:** MIT
- **Approach:** Quarto extension bundled inside the R package (`inst/extdata/`)
- **Dependencies:** `fs` (Imports), `quarto` (Suggests)
- **File rename:** `before-body.tex` → `preamble.tex` (matches actual purpose)
- **Example:** Shipped as `inst/examples/` instead of vignette (avoids Quarto vignette engine dependency)
- **lipsum removal:** `\usepackage{lipsum}` removed from preamble — demo-only package
